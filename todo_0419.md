# Claude-Style Unified Tool Contract Todo

基于 [design_0419.md](/Users/lizhongxuan/Desktop/aiops-codex/design_0419.md) 的实施清单。

目标口径：

- 参考 Claude Code 的 `Tool<Input, Output, Progress>` 思路，统一 `aiops-codex` 的工具契约
- 保留当前 `snapshot/cards/approvals/evidence/orchestrator` 读模型
- 不直接照搬 `ReactNode`，改为 **Vue-compatible display descriptor**
- 所有目标工具都要逐个核对：
  - 当前等价物是什么
  - 接口是否已迁移
  - prompt 是否完整迁移
  - 展示层是否已接入 descriptor
  - 仍存在哪些问题

---

## 总体原则

- 所有工具必须最终通过统一接口注册
- `description()/prompt` 必须有单一真源
- 权限、只读性、破坏性、并发安全性必须有结构化返回
- 前端展示必须消费结构化 display schema，而不是继续靠文本启发式猜工具
- 所有“迁移完成”的判断，必须同时覆盖：
  - 后端接口
  - prompt
  - projection
  - 前端展示
  - 测试

## 当前结论

这不是“把 Claude 的 Tool 类型抄过来”这么简单，而是一次：

- 统一工具执行契约
- 收回 prompt 真源
- 建立 descriptor 展示层
- 收缩前端启发式
- 保留 snapshot/card durable read model

的渐进式迁移工程。

真正的完成标准不是某个接口文件存在，而是：

- 每个目标工具都在台账里被核对过
- 每个目标工具的 prompt 都被单独校验过
- 每个目标工具的问题都被记录和闭环过
- 前后端展示都不再靠散落的启发式拼出来
