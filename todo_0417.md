## 分阶段验收

### Milestone A

范围：

- 完成 Phase 1-3
- bifrost 路径发事件但 UI 仍旧正常

验收标准：

- dispatcher + event bus + runtime/card/approval projection 可运行
- 现有核心测试不倒退

### Milestone B

范围：

- 完成 Phase 4-6
- dynamic tool 主路径迁移

验收标准：

- 动态工具路径不再直接手写大部分副作用
- 审批统一收口

### Milestone C

范围：

- 完成 Phase 7-8

验收标准：

- orchestrator phase 同步改为事件投影
- event store 能支撑调试与后续 replay

## 建议实施顺序

建议按这个顺序推进：

1. Phase 1
2. Phase 2
3. Phase 3
4. Phase 5
5. Phase 4
6. Phase 6
7. Phase 7
8. Phase 8

原因：

- 先建立事件和投影骨架
- 再先迁 bifrost 路径，风险更可控
- 审批统一要放在 dispatcher 稳定后
- dynamic tool 最杂，适合后迁
