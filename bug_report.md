# Bug Report

## 2026-04-01 协作工作台真实功能测试

### 1. 空态页“新建工作台”按钮点击无反应
- 环境：`http://127.0.0.1:18085/protocol`，本地前端 dev server，后端 `127.0.0.1:8080`
- 复现步骤：
1. 打开协作工作台页面 `/protocol`
2. 页面处于“当前不是协作工作台会话”空态
3. 点击空态卡片中的 `新建工作台`
- 实际结果：
  点击后页面没有任何变化，仍停留在空态页，没有创建或切换到新的 workspace 会话，也没有错误提示或 loading 反馈。
- 期望结果：
  点击后应立即创建一个新的协作工作台会话，或者至少展示明确的 loading / 错误反馈。
- 影响：
  用户无法从空态页进入协作工作台主流程，主路径被阻断。
- 备注：
  该问题已通过真实浏览器操作复现，不是单纯的单元测试问题。

### 2. 工作台空态页附近多个主按钮缺少可见反馈
- 环境：`http://127.0.0.1:18085/protocol`
- 复现步骤：
1. 进入协作工作台空态页
2. 依次点击 `新建会话 空白`、左侧导航 `协作工作台`、顶部 `终端`
- 实际结果：
  点击后页面没有出现明显切换、弹层、loading 或错误提示，用户难以判断是按钮失效、请求失败，还是页面仍在处理中。
- 期望结果：
  主按钮至少应有明确反馈，包括页面切换、抽屉展开、loading、toast 或错误提示。
- 影响：
  空态页附近的主操作可用性不明确，用户很容易误判为页面卡死。

### 3. 协作工作台页面加载时存在前端 Console Error
- 环境：`http://127.0.0.1:18085/protocol`
- 现象：
  Playwright snapshot 连续显示页面存在 `1 errors, 0 warnings` 的 console 状态。
- 当前影响：
  具体错误内容还未提取，但它与页面多处交互无反馈同时出现，值得优先排查。
- 建议：
  后续排查时先抓取浏览器 console 明细，确认是否有运行时异常阻断了工作台空态页交互。
